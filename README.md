# Data Immersion & Wrangling

## 📌 Project Overview

This project focuses on the **first and most important stage of data analysis: Data Immersion and Data Wrangling**.
The objective is to understand a dataset, evaluate its quality, clean inconsistencies, and prepare it for further analysis.

In real-world data projects, raw data often contains **missing values, duplicates, inconsistent formats, and outliers**.
This project demonstrates how to identify and resolve these issues using data processing techniques.

---

## 🎯 Objective

The main goals of this project are:

* Understand the structure and meaning of the dataset
* Create a **data dictionary** describing each variable
* Identify **data quality issues**
* Clean and transform the dataset
* Produce a **final analysis-ready dataset**

---

## 📊 Steps Performed

### 1️⃣ Data Access & Familiarization

* Loaded the dataset
* Explored dataset structure (rows, columns, data types)
* Understood the purpose of each variable
* Created a **Data Dictionary** describing variables

---

### 2️⃣ Data Quality Assessment

Initial data profiling was performed to detect:

* Missing values
* Duplicate records
* Inconsistent formatting
* Outliers

This step helps understand the overall **data quality** before cleaning.

---

### 3️⃣ Data Cleaning & Transformation

Data cleaning scripts were written using tools like **Python (Pandas) / SQL** to fix issues such as:

* Handling missing values
* Removing duplicates
* Standardizing formats (dates, text fields)
* Correcting inconsistent values
* Creating new features

Example:

* Created new columns such as **Customer Age** from **Date of Birth**
